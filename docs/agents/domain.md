# Domain Docs

This repository uses a single-context layout.

## Source of truth

- root glossary: `CONTEXT.md`
- architecture decisions: `docs/adr/`
- product and technical review drafts: `docs/draft-review/`

## Rules

1. Read `CONTEXT.md` before introducing or renaming domain concepts.
2. Update `CONTEXT.md` when domain terminology changes.
3. Create an ADR only for decisions that are hard to reverse, surprising without context, and based on a real trade-off.
4. Keep implementation details out of `CONTEXT.md`.
5. Treat draft review docs as planning inputs, not as the glossary.
